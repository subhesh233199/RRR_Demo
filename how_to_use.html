How to Use the RRR Release Analysis Tool

Overview

The RRR Release Analysis Tool is an AI-powered web application for automated analysis of software release readiness PDF reports. It extracts, analyzes, visualizes, and summarizes key release metrics for two products: TM and WST.

  • AI-generated executive and brief summaries
  • Trend charts for all key metrics
  • One-click editing and downloading of reports
  • Caching for quick repeated analysis
  • Modern, browser-based interface

Step 1: Prerequisites

  • Python 3.12.4 (recommended)
  • All dependencies installed: 
    pip install -r requirements.txt
  • A valid Azure OpenAI API key in a .env file
  • Your PDF release readiness reports, saved in a folder

Step 2: Start the Backend

  1. Open a terminal in your project directory.
  2. Run: 
    python main.py
  3. The FastAPI server should start at http://127.0.0.1:8080 (watch your terminal for confirmation).

Step 3: Open the Web Frontend

  1. Open frontend.html in your browser (Chrome, Edge, or Firefox recommended).
  2. The UI has three main tabs: Home, Brief Summary, and Executive Summary.

Step 4: Run an Analysis

  1. Select Product: Use the dropdown to choose TM or WST.
  2. Enter Folder Path: Paste the path to your PDF reports folder.
    Example: C:\Users\YourName\Desktop\ReleaseReports
  3. (Optional) Clear Cache: Check this if you want to force a fresh analysis.
  4. Click "Analyze": Press the blue Analyze button. The app will display a spinner:
    “Analyzing PDFs… This may take a few minutes.”

Step 5: Review Results

  • Executive Summary Tab:
    • Analysis Report: Full markdown report (Overview, Metrics Summary, Key Findings, Recommendations)
    • Visualizations: Trend charts and grouped bar charts (all key metrics)
    • Evaluation: AI judge feedback and score
    • Hyperlinks: Clickable links found in the PDFs
    • You can:
      - Edit the report
      - Download as HTML or PDF
  • Brief Summary Tab:
    • Concise summary of key findings
    • Extracted hyperlinks
    • Edit/download like the main report

Step 6: Additional Features

  • Caching: Avoids reprocessing unless files change or cache is cleared.
  • Multi-product: Supports TM and WST releases (ensure correct selection).
  • Metrics Coverage: All required metrics and table formats for both products are handled per config.
Troubleshooting:
No Results? Check your folder path and that there are at least 2 PDF files.
Wrong Product? Ensure your selection matches your PDF format.
Backend Not Running? Start main.py and check the terminal for errors.
Cache Issues? Use "Clear Cache" for a fresh run.

FAQ

  • Q: What PDF formats are supported?
    A: TM and WST release readiness PDFs using your org’s templates.
  • Q: Where are generated charts saved?
    A: In a visualizations/ folder (backend); shown as images in the UI.
  • Q: Can I modify the summaries?
    A: Yes! Both summaries are editable before you download them.
Summary Flow:
1. Start backend → 2. Open frontend.html → 3. Select product, enter folder path, Analyze → 4. Wait → 5. View/edit/download results in tabs
